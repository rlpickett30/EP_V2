EnviroPulse V2.1 - Node Event Contract
========================================

Scope: Node-side event flow only.
Status: Code-aligned draft after installer/node-runtime refactor.
Boundary: This contract stops at the node's outbound message. Server-side promoted events such as SERVER_AVIS_LITE, SERVER_GPS_COORD, NODE_STATE_UPDATED, and NODE_TDOA_STATE are downstream server contracts, not node events.

Standard Event Envelope
-----------------------

Every canonical node event should use this envelope:

event_type: string
source: string
target: string
timestamp: UTC ISO string
payload: dictionary

The node event bus supports both forms:

event_bus.publish(event)
event_bus.publish(event_type, event)

target is a routing intention, not proof of delivery. Common node targets are:

server
sender
birdnet
node
journal

Direction Groups
----------------

Server -> Node:

TDOA_REQUEST
SEND_NODE_CHANGE_MODE

Node-local only:

RECORDING_AVAILABLE

NETWORK_DISCONNECTED

NODE_REGISTER

Event Contracts
===============

TDOA_REQUEST
------------

Purpose:
Server command requesting a synchronized TDOA recording from one or more nodes.

Direction:
Server -> Node local bus.

Node publisher:
communication_dispatcher.py, after inbound listener decode.

Original platform source:
Server TDOA subsystem.

Node subscribers:
- microphone_dispatcher.py
- journal_dispatcher.py

Expected result:
TDOA_REQUEST
-> Microphone
-> TDOA_RECORDING
-> Communication
-> Server

Required envelope fields:
event_type = "TDOA_REQUEST"
source = "server"
target = node_id, "node", "all", or "broadcast"
timestamp
payload

Communication-added payload fields:

SEND_NODE_CHANGE_MODE
---------------------

Purpose:
Server command instructing the node to change Communication transport mode.

Direction:
Server -> Node local bus.

Node publisher:
communication_dispatcher.py, after inbound listener decode.

Node subscribers:
- communication_dispatcher.py
- journal_dispatcher.py

Required command meaning:
One of the supported mode fields must resolve to Wi-Fi or LoRa.

Supported fields:
mode
network_mode
transport
command
action
value
payload.mode
payload.network_mode
payload.transport
payload.command
payload.action
payload.value

Accepted Wi-Fi aliases:
wifi
wi-fi
udp
enable_wifi
send_wifi
send_via_wifi
wifi_on

Accepted LoRa aliases:
lora
lorawan
enable_lora
send_lora
send_via_lora
lora_on

Expected result:
SEND_NODE_CHANGE_MODE
-> Communication mode handler
-> active_transport updated
-> NETWORK_CONNECTED or NETWORK_DISCONNECTED
-> optional queue flush

Atlas note:
Keep the name SEND_NODE_CHANGE_MODE for now. It is awkward on the node side, but current code uses it. Rename only after a full server/GUI scan.

Atlas Notes for Future Work
===========================

RECORDING_AVAILABLE should remain node-local because it feeds BirdNET. TDOA_RECORDING is the server-facing recording response.

GPS_COORD is both a server-facing coordinate event and a local BirdNET location update. Do not remove BirdNET from that path.

RTK_STATE must stay role-aware. Base readiness is about RTCM output/transport. Rover readiness is about received corrections and fix quality.

NETWORK_CONNECTED and EVENT_SENT should not be treated as server acknowledgements. They are node-side Communication state and send-handoff events.

SEND_NODE_CHANGE_MODE should stay unchanged until the server and GUI contracts are scanned. Rename pressure is real, but premature renaming would create unnecessary breakage.

MICROPHONE_SYNCED is now one of the most important proof events for the current TDOA milestone. Journal should subscribe to it.
