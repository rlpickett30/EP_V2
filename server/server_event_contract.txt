============================================================
HEADING STANDARD
================

This document records only events that are currently known to exist in the running server system or current verified server event drawing.

Each event contract must include:

1. Publication event name.
2. Publishing subsystem.
3. Payload format expected.
4. Current subscribers.
5. Description of publication.

Contract rules:

1. Add only verified events.
2. Do not document planned events as active contracts.
3. Do not rename an event in this document unless the implementation has also changed.
4. If an event is converted before transmission, preserve both the source event name and the server-normalized event name.
5. Payload fields must match currently observed journal payloads.
6. Unknown payload fields must be marked as unknown, not guessed.
7. Unknown subscribers must be marked as unknown, not guessed.
8. Server-side behavior must be documented only when shown by server journal evidence, server code, or the current server event timeline.
9. EVENT_SENT is treated as a communication confirmation event, not as the original event being sent.
10. Registry-owned publications are treated as server-normalized events.
11. NODE_TDOA_STATE is created by platform_registry_TDOA_manager.py and is separate from NODE_STATE_UPDATED.
12. NODE_STATE_UPDATED is a GUI-facing registry state event.
13. NODE_TDOA_STATE is a TDOA-facing registry state event.
